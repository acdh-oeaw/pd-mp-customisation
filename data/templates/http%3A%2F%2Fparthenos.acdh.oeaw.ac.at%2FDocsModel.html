Parthenos Entities Model

Being worked on. See PD User Texts gdoc.

These pages informs about the data model underlying the application

The PARTHENOS Entities Model (PEM) is a common semantic model developed in the PARTHENOS project which extends the CIDOC CRM. The PEM is described in further depth in the following document: PARTHENOS D5.5 Report on the Common Semantic Framework. It is designed to be flexible enough to map the many different types of metadata schemas used throughout the Humanities uniformly.

Even though altogether many specific classes are introduced in the overall model, these can be largely categorised under the following main entities:

  • Actor
  • Dataset
  • Software
  • Service
  • Project
  • (Place)

Within the PEM a subset of properties has been identified as so called ‘minimal metadata’. “Such minimal metadata requirements are not meant as a ‘core’ standard but are rather specified precisely to demand the least possible data from aggregated sources while still being able to guarantee a functional management of the data being aggregated.” [D5.5] It is collected with regards to the entities referred to in the PEM for the PARTHENOS Joint Resource Registry (JRR) and allows the identification of distinct datasets, software and services.

Also, it is vital that datasets can be attributed to the appropriate actors responsible for these objects. The metadata set is kept as light as possible with the goal that all objects represented in the registry are traceable and that their provenance is accounted for.

In the overall dataset more information about the described entities is available and can be accessed via the search facilities described here.

Application

PARTHENOS Discovery is a customisation of the semantic knowledge management platform Metaphactory. It operates solely on top of a SPARQL endpoint and allows end users to visualize and explore the data in a user-friendly, highly customisable manner. PARTHENOS Discovery is considered complementary to the Joint Resource Registry, in that both operate on - and give access to - the same dataspace.

Further information: “metaphactory: A Platform for Knowledge Graph Management” http://www.semantic-web-journal.net/system/files/swj1950.pdf

Read about the main features of the application: A comprehensive description about the customisations of Metaphactory and considerations/findings about the tools for resource discovery in general can be found in the deliverable PARTHENOS D6.4 Report on services and tools (final). There is also a comprehensive documentation of the functionality Metaphactory provides.
