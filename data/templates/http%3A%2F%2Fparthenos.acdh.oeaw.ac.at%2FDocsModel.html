PARTHENOS Entities Model

This page informs about the data model underlying the application

The PARTHENOS Entities Model (PEM) is a common semantic model developed in the PARTHENOS project which extends the CIDOC CRM. The PEM is described in further depth in the following document: PARTHENOS D5.5 Report on the Common Semantic Framework. It is designed to be flexible enough to map the many different types of metadata schemas used throughout the Humanities uniformly.

Within the PEM a subset of properties has been identified as so called 'minimal metadata'. "Such minimal metadata requirements are not meant as a 'core' standard but are rather specified precisely to demand the least possible data from aggregated sources while still being able to guarantee a functional management of the data being aggregated." [D5.5] It is collected with regards to the entities referred to in the PEM for the PARTHENOS Joint Resource Registry (JRR) and allows the identification of distinct datasets, software and services. Also, it is vital that datasets can be attributed to the appropriate actors responsible for these objects. The metadata set is kept as light as possible with the goal that all objects represented in the registry are traceable and that their provenance is accounted for.
